# Розмір проєкту та деплой через GitHub

## Чому папка на диску велика (~400 MB)?

- **node_modules** (~398 MB) — залежності npm. Потрібні тільки локально для збірки та запуску. **Не заливаються в Git** (додано в `.gitignore`).
- **.next** (з’являється після `npm run build`) — кеш збірки Next.js. Теж ігнорується Git’ом. Netlify сам робить `npm run build` при деплої.

## Що реально летить у GitHub?

Тільки те, що не в `.gitignore`:

- `src/` — код
- `public/` — шрифт + картинки (лише використані: ScoopyCap.webp тощо)
- `Docs/`, конфіги, `package.json`, `package-lock.json`

**Орієнтовно кілька мегабайт** — зручно заливати через GitHub.

## Після клону або на новій машині

```bash
cd scoopy_sleep
npm install
npm run build   # опційно, для локальної перевірки
```

Netlify при деплої сам виконує `npm install` і `npm run build`.

## Якщо знову з’явиться .next

Після локального `npm run build` папка `.next` знову займе ~100+ MB на диску. Це нормально; у Git вона не потрапляє. Можна сміливо видаляти її для звільнення місця: `rm -rf .next`.
